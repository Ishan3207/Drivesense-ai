# DriveSense AI

**DriveSense AI** is a comprehensive, proactive vehicle health and geofenced safety ecosystem. It consists of a real-time mock OBD-II simulator backend to broadcast telemetry data and a React Native mobile application providing a dynamic **modern** dashboard with diagnostics powered by Advanced AI.

cd Drivesense-ai
docker-compose up --build

## Features & Program Functions

### 1. Mock OBD-II Simulator & Telemetry Broadcasting
- **Function:** Broadcasts realistic vehicle data—like speed, RPM, coolant temperature, engine load, and diagnostic trouble codes (DTCs).
- **Implementation:** Written in Python within the FastAPI backend (`drivesense-backend/app/services/obd_simulator.py`). The simulator acts as an asynchronous background loop generating data to replicate connected car hardware, pushing the live metrics to WebSockets.
- **Utility:** It removes the need for actual OBD-II hardware (e.g., ELM327 bluetooth scanners) during the development and demonstration phases, allowing developers and product teams to test real-time data flow perfectly.

### 2. Live Telemetry Dashboard
- **Function:** Displays the live metrics generated by the backend with a modern user interface.
- **Implementation:** Built using a mobile-first architecture utilizing React Native and the Expo framework (`drivesense-app`). It connects seamlessly to the backend via an open WebSocket connection, processing live data using `zustand` for high-performance state management across the app UI components.
- **Utility:** Gives the driver or operator a beautiful, unified view of vehicle health metrics and ensures data is accessible seamlessly in real time.

### 3. AI Vehicle Diagnostics
- **Function:** Ingests active OBD-II Diagnostic Trouble Codes (DTCs) and explains the issue in clear, easy-to-understand language.
- **Implementation:** Backed by integrations with the OpenAI or Google Gemini large language models. The backend creates API endpoints (`/api/v1/diagnostics`) to structure prompts dynamically based on real engine data metrics.
- **Utility:** Saves time and money by empowering regular drivers to understand "Check Engine Data" instantly, removing language barriers commonly associated with technical automotive mechanics.

### 4. Geofencing & Safety Zones
- **Function:** Tracks vehicle location using GPS coordinates to determine whether the car is entering or exiting sensitive zones like school or hospital areas.
- **Implementation:** Implemented with Expo Location in the mobile app to capture coordinates in both the foreground and background. On the backend, algorithms evaluate proximity to predefined polygons or geofenced bounds using geometry APIs and libraries (`geopy`).
- **Utility:** Enforces speed limits programmatically and proactively alerts drivers to decrease speeds, potentially increasing safety in critical communities.

### 5. Driver Scoring & Analytics
- **Function:** Analyzes driving patterns (e.g., hard braking, rapid acceleration) and creates an aggregated safety score.
- **Implementation:** Leverages PostgreSQL using SQLAlchemy models. The backend calculates infractions and normal driving events over time and calculates a rolling safety score stored continuously in the DB schema.
- **Utility:** Extremely beneficial for insurance modeling, fleet tracking, and encouraging better driving behavior over time.

## Technical Stack (Everything Used)

**Backend:**
- **FastAPI:** High performance asynchronous API framework for Python.
- **PostgreSQL & Asyncpg:** Relational database and high-performance asynchronous driver.
- **SQLAlchemy & Alembic:** Advanced ORM for handling data schemas and migrations securely.
- **WebSockets:** For achieving bidirectional, low-latency live streaming.

**Frontend:**
- **React Native & Expo:** Cross-platform framework for building the iOS, Android, and Web dashboards.
- **Zustand:** Highly optimized, minimal state management.

## Setup & Running using Docker

Running the application using Docker is straightforward. It provisions the database, the backend APIs, and the Expo development web server dynamically without installing any dependencies locally.

### 1. Clone the repository and configure environment
Change your working directory to the project's root:
```bash
cd drivesense
```

You can customize your API keys (e.g., `OPENAI_API_KEY`, `GOOGLE_GEMINI_API_KEY`) within `docker-compose.yml` under the `backend` service section, or create a `.env` file within `drivesense-backend/`.

### 2. Build and Start the Containers
Simply use `docker-compose` to spin everything up:
```bash
docker-compose up --build
```
This command performs the following:
1. Pulls the PostgreSQL 15 image and boots up a local database accessible on `5432`.
2. Builds the Python Backend container and exposes the APIs on `http://localhost:8000`. Inside, it securely bootstraps the tables on startup and automatically runs the OBD-II simulator loop.
3. Builds the Node Frontend container and triggers the Expo Web application server, hosting your UI on `http://localhost:8081`. 

### 3. Access the Applications
- **Frontend Dashboard (Web):** Visit [http://localhost:8081](http://localhost:8081) in your browser.
- **Backend Swagger API Docs:** Visit [http://localhost:8000/docs](http://localhost:8000/docs) to test API endpoints interactively.
- **Read Live Telemetry (WebSocket):** WSS streaming is open locally on `ws://localhost:8000/api/v1/ws/telemetry`.

### Stopping the execution
To gracefully stop your locally running instances:
```bash
docker-compose down
```
_Note: Database structures continue to persist cleanly across executions via the named `postgres_data` volume._
